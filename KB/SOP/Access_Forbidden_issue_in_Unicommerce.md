# Access Forbidden issue in Unicommerce:

Source: https://sites.google.com/shiprocket.com/sop-shiprocket/access-forbidden-issue-in-unicommerce

Search this site Embedded Files Skip to main content Skip to navigation Access Forbidden issue in Unicommerce: We have observed that multiple tickets are being raised, and sellers are getting "Access forbidden" in Unicommerce. This error is occurring because the seller is using a registered email address instead of an API user. Sellers cannot use registered email addresses to generate tokens. We have implemented a check on our end to restrict token generation through registered email addresses. Any registered email address used to create a token will result in an "Access forbidden" error or a connection breakage error. Please ask the seller to create an API user in the panel and use those API user credentials on Unicommerce. The process to create an API user is mentioned in the link below: https://apidocs.shiprocket.in/#intro Please pass this message to your team so that if any issues are raised by the seller, they can guide to update the API user and credentials in Unicommerce after creating it in Shiprocket.
